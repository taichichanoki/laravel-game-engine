# データ駆動型ノベルゲームエンジン『霧の森から』

LaravelとJavaScriptで構築した、データ駆動型のノベルゲームシステムおよび、そのシナリオ・素材を管理するための専用管理画面です。

単にストーリーを再生するだけでなく、実務でのWeb開発を想定したセキュリティ対策や、データマネジメント機能を作成しました。

---

##  主な機能と技術的工夫

### 1. セキュリティ・キャッシュコントロール
- **戻るボタンによる機密ページ閲覧の防止：**
  管理者画面でログアウトした後にブラウザの戻るボタンを押しても、ブラウザのキャッシュによって管理画面の魚拓が表示されてしまうことを防ぐため、ミドルウェア（`NoCache`）を作成。
  `Cache-Control: no-store` 等のヘッダーを強制注入し、戻るボタンを押してもサーバーサイドで認証チェックが走るようにしました。

### 2. Seederエクスポート機能
- **ワンクリックでのデータコード化：**
  管理画面から、現在登録されているシーン、ステップ、選択肢、アセットをマスターデータとして**ボタン一つでLaravelのSeederクラス（PHPコード）として書き出してダウンロードする機能**を実装。
  チーム開発でのデータ共有や、本番環境へのデータ移行を（`php artisan migrate:fresh --seed`）で一瞬で可能にできるよう作成しました。

### 3.サーバー内ファイルの自動クリーンアップ
- **実ファイルとDBの連動：**
  管理画面から画像や音声などのアセットレコードを削除した際、データベースのデータ消去だけでなく、サーバー（`public`フォルダ内）にアップロードされている**物理ファイル本体も`unlink()`で自動連動削除**する仕組みを実装。
  サーバー内に参照元を失ったファイルが蓄積し続ける問題を根本から防止しています。

### 4.データベースリレーション設計
- **外部キー制約と連動削除：**
  ゲーム内のシーン、ステップ、選択肢のデータを適切にリレーション。
  管理画面から特定のシーンを削除した際、そこにぶら下がっているステップ、選択肢のデータや、プレイヤーのセーブデータのロード時に対応シーンが存在しないという事態にならないよう、データベースの外部キー制約およびコントローラー側での連動削除を徹底しました。

### 5.プレイヤーの行動による動的なゲーム分岐（JS）
- **ステータス連動型選択肢：**
  JavaScript側でプレイヤーの「エネルギー」「異形度」「好感度」をリアルタイム管理。
  データベース側で設定された出現条件（`max_alignment_required` などの上限・下限値）をJSが判定し、条件を満たしていない選択肢の除外を行っています。これにより、ストーリーを追加していくことで、直前の選択肢による分岐のみでなく、複数の選択肢を組み合わせた分岐を実装することができるようにしています。

## 使用技術（テックスタック）

- **Backend:** PHP 8.x / Laravel 11.x
- **Frontend:** HTML5 / CSS3 / JavaScript (Vanilla JS)
- **Database:** MySQL
- **Tooling:** Git / GitHub

## セットアップ・データ復元方法

本リポジトリをクローン後、以下のコマンドを実行することで、作成済みのサンプルデータ（1章分）が自動投入され、すぐにゲームおよび管理画面の動作を確認いただけます。

```bash
composer install
cp .env.example .env
php artisan key:generate
# ⚠️ .env のDB接続設定完了後
php artisan migrate:fresh --seed
```
